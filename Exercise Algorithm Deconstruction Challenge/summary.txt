#Prompt 1: Understand an Algorithm Through Step-by-Step Analysis

 It is a weighted scoring algorithm, which wokrs to turn a task’s properties into
  one final numeric score.
The higher the score, the more important the task is considered.
The algorithm begins with zero and then adds or subtracts points.

priority_scores = {
    TaskPriority.LOW: 10,
    TaskPriority.MEDIUM: 20,
    TaskPriority.HIGH: 40,
    TaskPriority.URGENT: 60
}
score += priority_scores.get(task.priority, 0)

{This gives every task a starting importance level based on its priority.
  Meaning:
    LOW tasks start low
    URGENT tasks start much higher
This is the strongest single factor in the scoring model.}

if task.due_date:
    days_until_due = (task.due_date.date() - datetime.now().date()).days
    if days_until_due < 0:
        score += 35
    elif days_until_due == 0:
        score += 20
    elif days_until_due <= 2:
        score += 15
    elif days_until_due <= 7:
        score += 10

{This increases the score when the task deadline is close or already passed.
    Meaning:
        overdue tasks get the biggest boost
        tasks due today also get a strong boost
        tasks due soon get a smaller boost
This section captures time pressure.}

if task.status == TaskStatus.DONE:
    score -= 50
elif task.status == TaskStatus.REVIEW:
    score -= 15

{This lowers the score for tasks that are already completed or near completion.
     Meaning:
         DONE tasks become much less important
         REVIEW tasks are slightly less urgent than active unfinished tasks
This section captures workflow stage.}

     ##Glossary of Domain Terms Used in the Application

     Task: A unit of work (e.g., "Fix bug") with attributes for tracking progress, urgency, and metadata. The core entity users create and manage.
     Task Status: An enum representing a task's current state in its lifecycle (e.g., TODO, IN_PROGRESS, REVIEW, DONE). Drives workflow and visibility.
     Task Priority: An enum indicating a task's urgency or importance (LOW to URGENT). Used for sorting and escalation.
     Workflow: The sequence of status changes (e.g., TODO → DONE), potentially with rules (e.g., REVIEW as a gate).
     Overdue: A task with a past due_date and incomplete status (not DONE). Affects filtering and alerts.
     Tags: User-defined strings for categorizing tasks (e.g., "project-alpha"). Flexible but prone to inconsistency.
